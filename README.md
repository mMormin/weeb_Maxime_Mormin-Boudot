# Weeb Showcase Site

## Description du projet

Dans ce projet, nous avons pour mission de développer les premières interfaces du site internet de l’entreprise **Weeb**. Celle-ci souhaite créer un site incluant une partie vitrine, un blog ainsi qu’un espace d’authentification.

Nous nous focaliserons sur la partie vitrine durant la première semaine. L’objectif est de développer les pages :

- Home
- Contact
- Login

Une maquette réalisée par le client (Weeb) avec ses designers internes est mise à disposition. Le client impose quelques contraintes techniques concernant les outils et langages à utiliser :

- React
- CSS ou Tailwind
- Git

Vous pourrez utiliser l’IDE de votre choix.

Nom du repository : `weeb_Maxime_Mormin-Boudot`

## 🧩 Project Structure

```
src/
├── components/       # Composants réutilisables
├── pages/            # Pages : Home, Contact, Login
├── assets/           # Images, logos, polices
├── App.tsx           # Composant principal avec router
├── main.tsx          # Point d’entrée
└── index.css         # Styles globaux (Tailwind ou CSS custom)
```

## ✅ Completion Criteria

- Toutes les sections demandées sont développées et fonctionnelles (Home, Contact, Login)
- Tous les éléments de la maquette sont présents
- Contraintes respectées : utilisation de React, react-router-dom, Framer Motion
- Responsive design correct (mobile, tablette, desktop)

### Code & Git

- Code propre et bien indenté
- Commentaires explicatifs quand nécessaire
- Conventions de nommage respectées (variables, commits)
- Git flow optimal : issue → branche dédiée → PR → validation → suppression branche

### UI & UX

- Respect du design Figma
- Animations et transitions fluides
- Interaction (hover, focus) bien gérée

### Collaboration

- Bonne communication entre développeur et client
- Ajout éventuel d’éléments supplémentaires validés par le client
